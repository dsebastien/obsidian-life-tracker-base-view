# 2026-06-10

## Implemented #90, #91, #92, #109, #105, #100 (same day, after the review below)

### #90 — Grid view: flush pending auto-saves before teardown

- `saveTimers` map now stores `{ timerId, file, definition }` instead of bare
  timer ids, so pending saves can be replayed.
- New `flushPendingSaves()` (replaces `clearAllTimers()`): clears each timer
  and calls `savePropertyImmediate` with the latest value from
  `currentValues`. Called at the top of `onDataUpdated` and in `onunload`.
- `savePropertyImmediate` gained `updateCooldown = true` param; the flush
  passes `false` so the data update from the flushed write isn't swallowed
  by the save cooldown (the editors are torn down anyway).
- Files: `src/app/view/grid-view/grid-view.ts`.

### #91 — Capture modal: validation gate + surfaced save failures

- `saveCurrentPropertyImmediate` now validates non-empty values via
  `frontmatterService.validate` before writing (mirrors grid view). Empty
  values still pass through — writing them clears the property.
- Write failures now show a Notice (capture modal AND grid view) and route
  through `log()` instead of `console.error`.
- Files: `property-capture-modal.ts`, `grid-view.ts`.

### #92 — Missing values are gaps, not zeros

- `chart-aggregation.utils.ts`: all four `?? 0` coercions removed.
    - `aggregateForChart` / `aggregateForOverlayChart`: null values skipped;
      periods with only empty entries yield `null` → Chart.js renders a gap
      (its default `spanGaps: false`). Labels are kept (showEmptyValues
      semantics preserved).
    - `aggregateForScatterChart` / `aggregateForBubbleChart`: valueless
      entries filtered upfront (no point/bubble at 0; valueless entries no
      longer inflate bubble radii).
- 6 regression tests added; `docs/configuration.md` updated.
- Heatmap path already skipped nulls — unchanged.

### #109 — prefers-reduced-motion + focus-visible

- New `prefersReducedMotion()` util (`src/utils/dom.utils.ts`).
- Confetti (both bursts) gated in `property-capture-modal.ts`.
- Chart.js: `animation: chartAnimation()` (false under reduced motion) in
  all five initializers (`chart-initializers.ts`). Read Chart.js
  configuration/animations docs first per AGENTS.md mandate.
- `styles.src.css`: new **unlayered** accessibility section (unlayered
  styles beat all `lt-` layers): `:focus-visible` outline for all `lt-`
  prefixed elements + `@media (prefers-reduced-motion: reduce)` disabling
  keyframe animations (pulse, spinner, carousel slide-in, shimmer) and
  zeroing transition durations.
- The explicit play-button chart animation was left running — it is
  user-initiated playback, not decorative motion.

### #105 — "Capture today" command

- New command id `capture-today`, name "Capture today".
- `findTodayNote()`: exact basename match on the daily pattern
  (`parseDateFromFilename`, `TimeGranularity.Daily`, `isSameDay`); most
  recently modified wins on duplicates; Notice when none found.
- Files: `capture-command.ts`; documented in README + docs/property-capture.md.

### #100 — Heatmap streak stats

- New `HeatmapStreakStats` type (`currentStreak`, `longestStreak`,
  `activeCount`) on `HeatmapData` (non-optional; empty data gets zeros).
- New `src/app/services/heatmap-streak.utils.ts` — `computeHeatmapStreaks`:
  calendar-aware consecutiveness via `getTimeKey(incrementDate(...))`
  (cells are sparse); active = value non-null and non-zero (consistent with
  #87 zero-as-absence rendering); current streak requires the trailing run
  to reach the current or immediately preceding period. 11 tests.
- Rendered as a stats row below the heatmap legend
  (`.lt-heatmap-streaks`), refreshed on in-place updates; cleaned in
  `destroy()`. Documented in docs/visualizations.md.
