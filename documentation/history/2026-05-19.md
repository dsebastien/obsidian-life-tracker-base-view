# 2026-05-19

## Heatmaps auto-scroll to the end on open

Heatmap data is rendered ascending (oldest left → newest right), but the
scrollable `.lt-heatmap` container (`overflow-x-auto`) opened pinned to the
left, hiding the freshest data behind a scroll on any view wide enough to
need it.

### What changed

- `HeatmapVisualization.render` now defers a `requestAnimationFrame` callback
  that sets `scrollLeft = scrollWidth` on the heatmap's outer scrollable
  element, so the rightmost (most recent) cells are in view on first paint.
- The pending frame handle is tracked on the instance and cancelled in
  `destroy()` and on subsequent renders to avoid leaks or stale writes after
  unload.
- Scroll-to-end only fires on full `render()` — not on the in-place `update()`
  path — so user scroll position is preserved when data refreshes without a
  structural re-render.

### Files

Modified:

- `src/app/components/visualizations/heatmap/heatmap-visualization.ts`
    - Added `pendingScrollFrame` field and `scrollToEnd(scrollEl)` helper.
    - Called `scrollToEnd` at the end of `render` against the `.lt-heatmap`
      element (which owns the horizontal overflow).
    - Cancel the pending frame in `destroy`.
