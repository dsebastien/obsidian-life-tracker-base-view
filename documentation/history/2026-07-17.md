# 2026-07-17

## Two-phase release workflow (attestation bound to tag commit)

Ported the fixed `release.yml` from `obsidian-remarkable-sync` (proven live with
its 1.5.0 release: Sigstore cert sha == tag commit). The old workflow built and
attested at the triggering commit, then created the version-bump commit and tag
— so the provenance attestation pointed at a different commit than the release
tag, and the community-catalog review rejected the release ("attestation failed
cryptographic verification").

### What changed

- `.github/workflows/release.yml` — replaced verbatim with the two-phase
  design: a `prepare` job (dispatch on a branch) bumps versions, generates the
  changelog, runs the build gate (`bun run build` + `bun test`), commits,
  pushes a lightweight tag, then re-dispatches the workflow at the tag ref
  with `publish=true`; a `publish` job (dispatch-at-tag or bare tag push)
  builds, attests (`continue-on-error` removed), and creates the release at
  `github.sha` == tag commit. Recovery paths documented in the workflow header.
- `scripts/release.sh` — updated the "The GitHub workflow will:" summary to
  describe the two-phase flow.
